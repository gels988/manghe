# [双轨制聊天室] 第一阶段任务排期表

## 战略目标
- 在现有主界面上，先打通双轨 MVP：
  - 人类通道：九宫格 + 房间号 + 随机盐 + 万维码数字串
  - 智能体通道：X402 支付握手 + 结构化 JSON 交换窗口

## 第一阶段排期
### D0 当日
- 梳理现有聊天室入口、主界面状态、信令依赖
- 在主界面增加智能体握手反馈窗口
- 输出网关双轨路由伪代码
- 输出智能体握手 JSON 规范
- 输出前端状态机与组件树

### D1
- 增加网关侧 Token 识别规则
- 定义 Human_Token / Agent_Token 最小字段集
- 将 402 paywall 响应接入前端控制台

### D2
- 打通本地预演链路：
  - 无 Token -> 402
  - 携带 Human_Token -> 人类聊天
  - 携带 X402 回执 -> Agent_Token
- 完成前端 Tab 切换与结构化握手面板联动

### D3
- 接入真实后端 API
- 加入握手日志、状态灯、失败回退路径
- 准备 A/B 本地联调

## 验收标准
- 主界面存在双轨窗口且不破坏原人类通道
- 机器握手 JSON 可在前端预演并产生结构化响应
- 设计文档全部归档在本资料夹
