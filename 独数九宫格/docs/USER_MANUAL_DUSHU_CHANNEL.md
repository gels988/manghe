# 独数九宫格通讯系统使用手册

适用范围: 当前主界面 `index.html/index.html`

文档用途: 面向最终用户，说明本系统如何输入、如何转换、如何恢复信息。

## 1. 系统定位

本系统表面是九宫格页面，实际使用时请以九宫格下方的三个输入区域为准。

九宫格的作用:

- 用于界面伪装
- 用于视觉干扰
- 不决定真实通讯通道

真正决定通讯是否一致的，是下面这三项:

1. 房间号
2. 随机盐
3. 输入内容

## 2. 三层加密体系

本系统可理解为三层保护，不需要理解内部算法，只需要按规则使用。

### 第一层: 基础层

- 你输入的原始文字内容
- 可以是中文、English、日本語、한국어
- 这是你想表达的真实信息

### 第二层: 通道层

- 通道由第一框中的房间号决定
- 用户只输入 `1-9` 单个数字
- 系统内部会自动映射成两位房间号:
  - `1 -> 01`
  - `2 -> 02`
  - `3 -> 03`
  - 依此类推

### 第三层: 随机盐

- 随机盐由用户自己输入
- 可以是文字，也可以是数字
- 例如:
  - `李白`
  - `111`
  - `f8618`
  - `abc`
- 随机盐相当于同一通道里的第二把钥匙

结论:

- 房间号相同
- 随机盐相同
- 才能正确互相还原内容

## 3. 主界面三个框的作用

## 第一框: 房间号 + 随机盐

你需要填写两项:

- 房间号: 输入 `1-9` 单个数字
- 随机盐: 输入任意你和对方约定好的文字或数字

注意:

- 九宫格上显示什么，不重要
- 九宫格有没有显示该数字，也不重要
- 真正起作用的是第一框里输入的房间号和随机盐

## 第二框: 原文输入框

这个框主要用来:

- 输入你想发送的原文
- 或者粘贴别人发来的 `ID+数字串`

当你输入原文时:

- 第三框会自动生成转换后的 `ID+数字串`

当你粘贴 `ID+数字串` 时:

- 系统会自动识别
- 自动恢复出原始文字

## 第三框: ID+数字串框

这个框主要用来:

- 显示系统生成的 `ID+数字串`
- 或者粘贴别人发来的原文或 `ID+数字串`

当你输入原文时:

- 这里会自动生成可复制的 `ID+数字串`

当你把 `ID+数字串` 粘贴到这里时:

- 第二框会自动恢复成原文

也就是说:

- 第二框和第三框可以互相转换
- 任意一边输入内容，另一边都会尽量自动还原

## 4. 正确使用流程

### 发送方 A 的操作

1. 在第一框输入房间号，例如 `3`
2. 在第一框输入随机盐，例如 `李白`
3. 在第二框输入你要发送的话
4. 第三框会自动生成 `ID+数字串`
5. 点击复制
6. 把复制结果发送给 B

### 接收方 B 的操作

1. 不需要先点九宫格
2. 可以直接把 A 发来的 `ID+数字串` 粘贴到第二框或第三框
3. 系统会自动识别内容
4. 系统会自动恢复出原文
5. 如果消息中带有对应通道信息，系统会自动切换到正确房间并回填随机盐

## 5. 最推荐的双机使用方法

为避免出错，建议 A 和 B 提前约定:

- 同一个房间号
- 同一个随机盐

推荐格式:

- 房间号: `3`
- 随机盐: `李白`

实际效果:

- A 输入 `3 + 李白`
- B 输入 `3 + 李白`
- 双方就进入同一真实通道

如果改成:

- A 是 `3 + 李白`
- B 是 `3 + 111`

那么虽然房间相同，但随机盐不同，恢复结果就不会一致。

## 6. 复制与重置

第三框下方通常会有几个按钮:

- `复制`: 复制当前生成的 `ID+数字串` 或当前有效结果
- `重置`: 清空第二框和第三框内容，重新开始
- `刷新频道`: 刷新九宫格伪装显示，不改变协议规则

第一框还带有:

- `清除盐值`: 只清空随机盐，便于重新约定

## 7. 九宫格的正确理解

九宫格现在只负责:

- 外观伪装
- 页面拟态
- 视觉混淆

九宫格现在不负责:

- 决定真实频道
- 决定通讯是否成功
- 决定 A 与 B 是否能互相识别

所以请记住一句话:

`真正的通讯通道，不在九宫格上，而在第一框里。`

## 8. 输入建议

为了提高稳定性，建议这样使用:

- 房间号尽量用单个数字 `1-9`
- 随机盐不要太短，建议 2 到 8 个字符
- 随机盐输入后，A 和 B 不要随意改动
- 复制 `ID+数字串` 时不要手动删改
- 粘贴时尽量完整粘贴整段内容

推荐随机盐示例:

- `李白`
- `青龙`
- `f8618`
- `111`
- `AIM`

不推荐:

- 空盐值
- 临时乱改
- 一边用中文，一边用另一组数字盐却以为相同

## 9. 常见情况

### 情况 1: 为什么九宫格没有显示某个数字?

答:

- 这是伪装效果
- 不影响真实通讯
- 你只需要在第一框手动输入房间号即可

### 情况 2: 为什么我输入了文字，没有生成结果?

请检查:

1. 是否已在第一框输入房间号 `1-9`
2. 是否已输入随机盐
3. 是否当前内容为空

### 情况 3: 为什么双方房间号相同，却恢复不一致?

请检查:

1. 随机盐是否完全一致
2. 是否有人多打了空格
3. 是否有人修改过复制出来的 `ID+数字串`

### 情况 4: 我应该把内容粘贴到第二框还是第三框?

答:

- 两边都可以
- 系统会自动判断
- 推荐习惯:
  - 输入原文时用第二框
  - 接收复制串时粘贴到第三框

### 情况 5: 为什么系统会自动切换房间或回填随机盐?

答:

- 因为收到的 `ID+数字串` 中本身带有对应上下文
- 系统会自动恢复通道信息，减少人工错误

## 10. 最短操作口诀

可记为:

`先定房间，再定盐；文字进第二框，数字串看第三框；收到串，任意框一贴就还原。`

## 11. 一组完整示例

### A 端

- 房间号: `3`
- 随机盐: `李白`
- 第二框输入: `今晚八点见`

系统会在第三框生成一段 `ID+数字串`

### B 端

- 直接把 A 发来的 `ID+数字串` 粘贴到第二框或第三框

系统会自动:

- 切换到对应内部通道
- 回填随机盐
- 还原出 `今晚八点见`

## 12. 最终原则

请始终按下面的顺序理解系统:

1. 九宫格只是伪装
2. 第一框决定真实通道
3. 随机盐决定第二把钥匙
4. 第二框和第三框负责互相转换
5. 不要手改生成后的 `ID+数字串`

以上规则记住后，系统就会稳定很多。
