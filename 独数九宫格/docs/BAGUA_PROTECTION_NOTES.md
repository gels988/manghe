# 八卦保护层说明

适用页面:

- `index.html/index.html`
- `juanzeng.html`
- `zixitong.html`

## 保护方式

- 页面 `head` 与外链脚本保持明文，保证浏览器可正常加载。
- 页面 `body` 主体改为八卦三爻编码串。
- 浏览器先加载 `js/bagua.protect.js`，把八卦串还原成真实 DOM，再执行原页面脚本。
- 编码规则使用固定映射:
  - `1 -> 111`
  - `2 -> 011`
  - `3 -> 101`
  - `4 -> 001`
  - `5 -> 110`
  - `6 -> 010`
  - `7 -> 100`
  - `8 -> 000`

## 本地重生成

在项目根目录执行:

```powershell
& 'd:\独数九宫格\tools\protect-with-bagua.ps1'
```

如果后续只想重封装单个页面，可给脚本传文件路径数组。

## 作用边界

- 能防: 直接查看源代码时一眼复制页面结构、低成本照搬页面骨架。
- 不能防: 熟悉前端的人在浏览器运行后查看还原后的 DOM、抓取已解码内容、继续逆向 JS。
- 结论: 这是“一层可运行保护壳”，用于提高模仿门槛，不是不可逆加密。

## A/B 本地联调前检查

- 打开三页时不应出现空白页。
- 主界面激活遮罩、三框互转、房间号与随机盐逻辑应正常。
- `juanzeng.html` 与 `zixitong.html` 页面按钮和列表应正常渲染。
- 若修改了这三页的 `body` 结构，记得重新执行八卦封装脚本。
