# [OPEN] debug-b-device-flow

## 症状
- B 电脑主界面过去无法打开，表现为闪退/打不开。
- B 电脑注册界面可以打开，但提示数据库未连接，无法注册。
- 期望：B 电脑能完成注册、激活并进入主界面使用层。

## 当前假设
1. 主界面闪退是因为 B 电脑首次加载时缺少某个本地依赖或脚本初始化顺序异常，导致页面在运行期抛错。
2. 注册页无法注册是因为 B 电脑没有本地 Supabase 配置，`db_client.js` 初始化失败后注册逻辑直接中断。
3. A/B 使用不同访问源（localhost / 局域网 IP）导致本地缓存与身份数据隔离，B 电脑没有拿到 A 电脑上的身份/配置。
4. 八卦保护壳在 B 电脑环境下解码后脚本绑定时机异常，导致主界面部分节点不存在或事件未绑定。
5. 注册页虽然能打开，但依赖的是云端配置而不是纯本地注册；当前缺少“未配置数据库时的降级路径”。
